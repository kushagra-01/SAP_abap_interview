# 00 Study Guide (Memorize Like a Dev)

## 7-day crash plan (repeat daily)
- **30 min**: MEMORY HACKs only (all topics)
- **45 min**: Q&A drilling (say answers out loud)
- **30 min**: code patterns (type, don’t just read)
- **15 min**: traps + “what would you do in prod?”

## Spaced repetition (simple)
- Day 1: learn
- Day 2: review
- Day 4: review
- Day 7: review
- Day 14: final review

## Answer formula (safe in interviews)
- **Definition** (1 line)
- **Where used** (1 line)
- **Key choice / trade-off** (1 line)
- **1 pitfall** (1 line)

## Core 80/20 checklist (must know)
- ABAP basics: types, Open SQL, work areas, exceptions
- Internal tables: standard/sorted/hashed + read patterns
- DDIC: table/structure/domain/data element, keys, buffering
- Modularization: forms/FM/methods, updates/commits
- OOP: interface, inheritance, polymorphism, events
- Performance: DB vs app server, indexes, FAE pitfalls
- CDS: annotations, associations, authorization, pushdown
- OData: entity sets, $filter/$expand, error handling
- RAP: behavior, EML, unmanaged vs managed, draft
